Railcast — current shipped state

Not a redesign pitch — a faithful reconstruction of what's actually in main right now, transcribed screen-by-screen from the Kotlin source (colours, strings, component contracts) rather than from the design docs' wireframes. Where the two disagree, this shows what's really there.

Board = live truth (dark, mono, flap-roll) Dashed underline = ESTIMATED (Confidence System) Brand blue = tappable only
The gap worth knowing about: the Journeys tab is still, almost byte-for-byte, the pre-redesign Home screen — including the Train / PNR toggle the design study's own Finding 7 argued against ("modes are where errors live"). The new single-input classifier only exists on Find. Saved-train cards on Journeys are a plain board (title + status + freshness) — no consequence line, no urgency ranking, no mini-spine; those only appear on the Track detail screen. Track/PNR and the ambient widget/notification are genuinely new and match the design docs closely. This prototype shows the real, current mix of both.

How this was built. Every string, colour token, and component contract above is transcribed from android/app/src/main/java/app/railcast/ as of the current main HEAD — HomeScreen.kt, ui/FindScreen.kt, feature/track/TrackScreen.kt, feature/pnr/PnrScreen.kt, feature/alerts/AlertsScreen.kt, feature/ambient/*, and the verified palette in docs/design/design-system.md §1.2. It is not run from the real APK (no device/emulator in this environment) — it's a faithful HTML reconstruction, not a build artifact. Try: type 129 or 4392 in Find to see the classifier morph; open Track to see the pinned board + dashed ~-estimate on the timeline; open PNR to see the masked pass.
